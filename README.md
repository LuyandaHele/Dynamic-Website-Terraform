# Dynamic-Website-Terraform
This project demonstrates the design and deployment of a highly available, scalable, and secure dynamic web application infrastructure on AWS using Infrastructure as Code (IaC) with Terraform. The architecture is deployed in the Amazon Web Services US East (N. Virginia) Region across multiple Availability Zones to ensure redundancy, fault tolerance, and high availability.

Project Overview

The infrastructure is provisioned and managed entirely through Terraform scripts executed locally using tools such as:

Visual Studio Code
Git
GitHub
AWS Command Line Interface

The project follows DevOps and cloud engineering best practices by automating cloud resource deployment, enabling version control, and supporting repeatable infrastructure provisioning.

Key Features
Multi-AZ deployment for high availability
Infrastructure as Code (IaC) using Terraform
Public and private subnet separation for improved security
Application Load Balancer for traffic distribution
Auto Scaling for dynamic resource management
NAT Gateways for secure outbound internet access from private instances
Highly available database architecture using Multi-AZ RDS
Monitoring and notification integration using SNS
GitHub integration for source control and collaboration

Project Objectives

The main objectives of this project are to:

Demonstrate cloud infrastructure automation using Terraform
Build a secure and scalable AWS environment
Implement a production-style multi-tier architecture
Apply DevOps and cloud engineering best practices
Improve fault tolerance, scalability, and availability


Conclusion

This project represents a real-world cloud infrastructure deployment that simulates how modern production-grade web applications are hosted in AWS environments. By leveraging Terraform and AWS services, the solution provides scalability, automation, security, and high availability while following industry-standard cloud architecture practices.
